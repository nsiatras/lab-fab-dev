# Αντιστοίχιση Συμβάσεων και Παραδοτέων

Ο φάκελος αυτός δημιουργείται για να λειτουργεί ως μητρώο ελέγχου ανάμεσα στις συμβάσεις, τα συμβατικά αντικείμενα και τα πραγματικά παραδοτέα που εντοπίζονται στα διαθέσιμα αρχεία του project.

Στόχος δεν είναι μόνο να καταγράφεται ποιος έλαβε αμοιβή και με ποια σύμβαση, αλλά να απαντάται το ερώτημα:

> Για κάθε σύμβαση ή συμβατικό αντικείμενο, υπάρχει αντίστοιχο παραδοτέο, τεχνική έκθεση, μελέτη, πλατφόρμα, αρχείο, URL ή άλλο τεκμήριο παραλαβής;

## Μεθοδολογία

Η αντιστοίχιση γίνεται σε τρία επίπεδα:

1. **Σύμβαση / ΑΔΑ / ποσό**: ποιο πρόσωπο ή ποιος ανάδοχος πληρώθηκε και με ποιο αντικείμενο.
2. **Αναμενόμενο παραδοτέο**: τι θα έπρεπε να υπάρχει με βάση το αντικείμενο της σύμβασης ή την περιγραφή της προκήρυξης.
3. **Εντοπισμένο παραδοτέο**: ποιο συγκεκριμένο αρχείο, URL ή τεκμήριο υπάρχει στα διαθέσιμα δεδομένα.

## Καταστάσεις ελέγχου

| Κατάσταση | Ερμηνεία |
|---|---|
| `Αντιστοιχισμένο` | Έχει εντοπιστεί συγκεκριμένο παραδοτέο και έχει συνδεθεί με τη σύμβαση. |
| `Μη εντοπισμένο` | Το αντικείμενο παραπέμπει σε συγκεκριμένο παραδοτέο, αλλά δεν έχει εντοπιστεί αντίστοιχο τεκμήριο στα διαθέσιμα αρχεία. |
| `Προς αντιστοίχιση` | Υπάρχει συμβατικό αντικείμενο, αλλά δεν έχει ολοκληρωθεί ακόμη ο έλεγχος παραδοτέων. |
| `Γενικό αντικείμενο` | Το αντικείμενο είναι συνεχής εργασία, επίβλεψη ή συμμετοχή σε πακέτα εργασίας και χρειάζεται περαιτέρω εξειδίκευση για να οριστεί ακριβές παραδοτέο. |

## Πρώτη κρίσιμη παρατήρηση

Στα διαθέσιμα αρχεία καταγράφεται σύμβαση της **Λυδίας Ντουσοπούλου** με αντικείμενο **«Μελέτη ψηφιακής πλατφόρμας για την κατασκευή ηλεκτρομηχανολογικών συσκευών με τρισδιάστατη εκτύπωση για εφαρμογές στον κλάδο της ναυτιλίας»** και αμοιβή **39.990,00 ευρώ**.

Στο παρόν project δεν έχει εντοπιστεί αντίστοιχη παραδοτέα μελέτη, ούτε τεκμηριωμένη ψηφιακή πλατφόρμα που να αντιστοιχεί ευθέως στο συγκεκριμένο αντικείμενο. Το σημείο αυτό καταγράφεται ως **μη εντοπισμένο παραδοτέο** και ως προτεραιότητα ελέγχου.

## Επόμενο βήμα

Το αρχικό μητρώο ξεκινά σε επίπεδο συμβατικού αντικειμένου ανά πρόσωπο. Επόμενη φάση είναι να σπάσει σε επίπεδο κάθε μεμονωμένης σύμβασης / ΑΔΑ, ώστε κάθε πληρωμή ή ανανέωση να συνδεθεί με συγκεκριμένο παραδοτέο ή με ρητή ένδειξη ότι δεν έχει εντοπιστεί.
